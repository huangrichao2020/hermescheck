# Mission

`hermescheck` is a 100-year open source project for understanding, auditing, and improving agent system design.

The mission is bigger than scanner rules:

- make agent failures discussable
- turn private operational pain into public method advances
- preserve signal across changing models, frameworks, and product eras
- keep doctrine, contracts, scanners, and governance evolving together

We assume the base model is often not the real problem. The wrapper architecture, memory policy, tool controls, output pipeline, and governance habits often decide whether an agent system becomes reliable or dangerous.

`hermescheck` therefore exists to help agents, humans, and maintainers do three things well:

1. Name real architecture failures precisely.
2. Verify claims with code-level or contract-level evidence.
3. Feed generalized lessons back into open source.

This project should be able to absorb 100 years of agent design shifts without losing clarity. That means:

- doctrine must remain explicit
- layers must remain separable
- contributions must be easy to generalize
- governance must protect quality without discouraging real-world feedback
